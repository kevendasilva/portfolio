Projetos / Gerenciamento de projetos

Gerenciamento de projetos

Sobre o projeto

O sistema foi construído para solucionar um problema interno, da empresa júnior GTi Engenharia Jr., da qual eu fiz parte. A GTi Engenharia Jr. é uma empresa júnior ligada aos cursos de Engenharia de Computação e Engenharia de Telecomunicações, da Universidade Federal do Ceará.

O problema consiste basicamente na organização e alocação dos membros, nos projetos da empresa. Com base nas informações do projeto, como a complexidade, tipo do projeto e o estado; é calculado a carga do projeto, para dado membro. A partir disso, é possível gerenciar a distribuição dos membros nos projetos, sem sobrecarregá-los.

Tecnologias

A principal tecnologia utilizada no desenvolvimento da aplicação, foi o Ruby On Rails. Além dessa tecnologia, baseada na linguagem Ruby, utilizou-se a linguagem JavaScript, em diversas áreas do sistema, como a visualização dos projetos, onde existe a visualização em quadro.

Desafios

À medida que o sistema foi ganhando novas funcionalidades, a empresa viu a possibilidade de vendê-lo para outras empresas juniores. No entanto, o sistema não havia sido modelado para lidar com a presença de usuários externos. Para solucionar tal problema, fiquei responsável pelo desenvolvimento de uma divisão no sistema (entre outras funcionalidades), que seria responsável pelo gerenciamento dos clientes (painel administrativo). A partir disso, é possível acompanhar melhor os clientes e verificar como está o plano de cada um.

Projetos / Casa do Idoso

Casa do Idoso

Sobre o projeto

Este foi um projeto de sistema web, criado para ajudar na gestão interna, da Casa do Idoso, de Limoeiro do Norte. Cada página (ou área) do sistema possui diversas informações, sejam elas a respeito dos residentes ou funcionários. Além da possibilidade de criação de relatórios, a partir de filtros e áreas do sistema; e diversas outras automações.

Tecnologias

A principalmente tecnologia utilizada para o desenvolvimento do sistema foi o framework Ruby On Rails. Além dos recursos fornecidos pelo framework citado acima, o sistema faz uso de três diferentes serviços. O sistema faz uso de três principais serviços, cada um responsável por determinada funcionalidade dentro do mesmo. Sendo eles: SendGrid API (para envio de e-mails para os usuários), Heroku (serviço usado para hospedagem da aplicação) e Google Cloud (para armazenamento de imagens e arquivos).

Desafios

Um dos desafios mais interessantes foi a criação da área de acompanhamento médico dos residentes. Nela o sistema, de forma automática, registra as movimentações de um dado residente (mudanças de medicamentos e etc.). Além disso, existe a possibilidade de adicionar registros de movimentações de forma manual.
